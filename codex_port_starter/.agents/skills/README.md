# Repo-Local Skills

Put project-specific Codex skills in this directory when a workflow is repeated
often enough to justify keeping it on disk.

Good candidates:

- multi-step review workflows used repeatedly in this repo
- deployment or release sequences with stable steps
- project-specific verification procedures
- packaging or reporting routines that benefit from one-call reuse

Avoid adding a skill too early. Start with `AGENTS.md`, plans, and templates,
then promote the workflow into a skill once it is stable.
